# The Words We Use — Jargon Decoder Tool

A self-contained tool that decodes and encodes organizational jargon using the formula system from *The Words We Use* by Jason Weimer.

## Just open it

**Open `index.html` in your browser by double-clicking.** The tool is fully self-contained and works from your desktop, a USB stick, or any folder — as long as you have run `sync_data.py` first to generate the companion data files.

If you are opening the tool for the first time from a folder (not a server), run this once:

```
python3 sync_data.py
```

Then open `index.html`. It will work offline from then on.

## What's in the box

```
jargon-decoder/
├── index.html                     ← Main tool page (double-click to run)
├── jargon-data.json               ← 983 jargon entries (source of truth)
├── jargon-data.js                 ← Auto-generated by sync_data.py
├── framework-standards-data.json  ← 78 frameworks & standards (source of truth)
├── framework-standards-data.js    ← Auto-generated by sync_data.py
├── sync_data.py                   ← Generates the .js files from the .json files
├── assets/
│   ├── style.css                  ← Black/white/gray/yellow design system
│   └── app.js                     ← Application logic
├── admin/
│   └── index.html                 ← Private page for adding new words
└── README.md                      ← This file
```

**The .json files are the source of truth.** Edit them, then run `sync_data.py` to regenerate the .js files.

**The .js files are generated.** Do not edit them directly — your changes will be overwritten the next time `sync_data.py` runs.

## How to deploy to your website

1. Upload the entire `jargon-decoder` folder to your server
2. The tool is live at the path you upload to (e.g. `jasonweimer.com/jargon-decoder`)
3. No build step. No server code. No dependencies.

The .js files must be present alongside `index.html` for the tool to work. Run `sync_data.py` before uploading if you have made changes to the JSON files.

## How it works offline (file:// mode)

When a browser opens `index.html` directly from a folder (not a server), `fetch()` cannot load the JSON files due to browser security restrictions. The tool solves this by loading data from companion `.js` files instead — these are included via `<script>` tags and set `window.JARGON_DATA` and `window.FRAMEWORKS_DATA` as global variables before the app runs. `sync_data.py` generates these .js files from the .json sources.

## Features

### Decode tab
- Search 983 jargon words to see formula, components, definition, what it really means, why the formula is shaped that way, and plain-language alternatives
- Alternative words that redirect to a primary entry show the redirect clearly, with a link to the full primary entry
- Words tagged as metaphors display a Metaphor badge
- Filter by cluster, formula type, status, or metaphors only
- Live search dropdown as you type
- Quick-pick buttons for common words

### Encode tab
- Enter up to 3 plain-language components
- Pick a formula type and/or cluster (optional)
- Returns top 3 jargon word matches ranked by component alignment
- Searches only entries with real formulas (not redirect alternatives)

### Browse tab
- All 983 words organized by cluster
- Color-coded status dots: black = core, yellow = extended, gray = alternative
- Gold dot = metaphor indicator
- Redirect alternatives show their primary target instead of formula type
- Filter by status or metaphors only

### Frameworks & Standards tab
- 78 named frameworks, accounting standards, and professional tools
- Separate from the jargon clusters — these have fixed meanings, not formulas
- Each entry shows: acronym expansion, origin/creator, definition, components, used-for, and related jargon terms (clickable — jump to full decode entry)

### Health Check tab
- Interactive Organizational Health diagnostic using the master formula
- Quick version (5 rapid questions + multiplier)
- Deep version (each component explained in detail)
- Score out of 80 with bar charts and a tailored diagnosis

### About tab
- The six formula types with generic templates
- The five clusters with their guiding questions

## Adding new jargon words

### Workflow
1. Open `admin/index.html` in your browser
2. Fill in the form
3. Click "Generate JSON" and copy the output
4. Open `jargon-data.json` and paste the new entry into the `entries` array (add a comma after the previous entry)
5. Run `python3 sync_data.py` from the jargon-decoder folder
6. Re-upload `jargon-data.json`, `jargon-data.js`, and `assets/app.js` to your site

### Core or extended entry structure
```json
{
  "word": "Your Word",
  "cluster": "Direction | People | Resources | Process | Results",
  "status": "core | extended",
  "formula_type": "Additive | Combination | Exponential | Multiplied | Refinement | Ratio",
  "formula": "Your Word = component1 + component2",
  "components": ["component1", "component2"],
  "definition": "Official dictionary-style definition.",
  "what_it_really_means": "The honest version.",
  "why": "Why the formula is shaped this way.",
  "alternatives": ["Plain language alternative 1", "Plain language alternative 2"],
  "is_metaphor": false
}
```

### Redirect alternative entry structure
```json
{
  "word": "Sprint",
  "cluster": "Process",
  "status": "alternative",
  "primary": "Agile Methodology",
  "formula_type": null,
  "formula": "See: Agile Methodology",
  "components": [],
  "definition": "A time-boxed iteration within an Agile project.",
  "what_it_really_means": "The most visible unit of Agile in practice.",
  "why": "Points to Agile Methodology. A sprint is the core delivery cycle.",
  "alternatives": [],
  "is_metaphor": false
}
```

## Metaphor notation

Words tagged with `"is_metaphor": true` display a Metaphor badge in their detail view and a gold dot in the Browse tab. A word qualifies as a metaphor when its organizational meaning is borrowed from another domain — sports, nature, military, medicine, navigation, physics, etc. Examples: Bottleneck (physical bottle neck), Runway (aircraft runway), Sprint (athletic sprint), Viral Growth (biological virus).

## Notation system

- **Generic formula types** in the About tab use `j = x + y + z` as the placeholder
- **Specific word formulas** use the actual word: `Vision = purpose + ambition + (time + possibility)`
- The letter `j` stands for jargon — the word being decomposed in the generic template
